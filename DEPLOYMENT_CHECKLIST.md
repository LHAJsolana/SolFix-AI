# SolFix AI Deployment Checklist

## Vercel

1. Create a Vercel project from the SolFix repository.
2. Set build command: `npm run build`.
3. Set install command: `npm install`.
4. Set production URL in `NEXT_PUBLIC_APP_URL`.
5. Deploy only after these commands pass locally:
   - `npm run db:generate`
   - `npm run lint`
   - `npm run typecheck`
   - `npm run test`
   - `npm run build`

## PostgreSQL / Supabase

1. Create a PostgreSQL database. Supabase is supported through a standard pooled `DATABASE_URL`.
2. Set `DATABASE_URL` in Vercel.
3. Run Prisma migrations against the production database before the public demo:
   - `npx prisma migrate deploy`
   - `npx prisma generate`
4. Verify `/api/health` reports `database: "postgres_configured"`.
5. Analyze a demo transaction, open its report URL, restart the deployment, and reopen the same report URL.

## RPC Configuration

1. Set `SOLANA_DEVNET_RPC_URL` for server-side devnet reads and Memo preparation.
2. Set `NEXT_PUBLIC_SOLANA_DEVNET_RPC_URL` for browser wallet confirmation.
3. Set `SOLANA_MAINNET_RPC_URL` for production mainnet reads.
4. Avoid public RPC endpoints for grant review traffic if possible.
5. Verify `/api/health` reports at least one reachable RPC provider.

## Optional AI Providers

Deterministic mode works without keys.

OpenAI:

```text
ENABLE_AI_EXPLANATIONS=true
AI_PROVIDER=openai
OPENAI_API_KEY=<secret>
OPENAI_MODEL=gpt-4o-mini
```

Anthropic:

```text
ENABLE_AI_EXPLANATIONS=true
AI_PROVIDER=anthropic
ANTHROPIC_API_KEY=<secret>
ANTHROPIC_MODEL=claude-3-5-haiku-latest
```

Never expose provider keys through `NEXT_PUBLIC_` variables.

## Wallet Memo Attestation

1. Use a browser wallet that exposes `window.solana`.
2. Fund the wallet with devnet SOL.
3. Open a saved report.
4. Click `Connect wallet`.
5. Click `Anchor Memo`.
6. Confirm the transaction in the wallet.
7. Confirm the saved Explorer link includes `cluster=devnet`.
8. Memo content must match `SOLFIX|v1|report:<reportId>|source:<shortSignature>|hash:<sha256>|ts:<unix>`.

## Metadata And Public Files

- `robots.txt` is generated by `src/app/robots.ts`.
- `sitemap.xml` is generated by `src/app/sitemap.ts`.
- Open Graph metadata is configured in `src/app/layout.tsx`.
- Add a favicon before a public marketing launch if brand polish matters.

## Health Checks

Check:

- `/api/health`
- `/`
- `/analyze`
- `/demo`
- one `/analysis/<id>` generated from demo mode
- one `/report/<slug>` generated from the same analysis

## Fallback Behavior

If `DATABASE_URL` is not set, SolFix uses local JSON report storage at `.data/solfix-reports.json`. This is useful for a real local tool, but serverless production should use PostgreSQL because local filesystem writes are not durable across instances.

## Manual Verification Before Grant Review

- Live PostgreSQL migration against a clean database.
- One real failed devnet transaction analysis.
- One successful transaction analysis.
- Wallet rejection and retry on a real browser wallet.
- Devnet Memo content on Solana Explorer.
- Production RPC plan and provider limits.
